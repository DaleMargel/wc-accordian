Accordian

An accordian expands to show details when clicked.
  • Header and details are user defined
    ... in fact this section header is an accordian
  • Can be expanded by clicking or setting open="true"
  • Can work independently or in group (one open at a time)
  • Code is tiny 1.4k w/o compression

These are grouped: only one of these can be open at any time

ApproachComments
angularThe heaviest and slowest of all web development frameworks.
vue?
svetleGreat idea, small sized application, but code needs to be compiled.
react Has some great bits, but takes on much of the work done by a modern browser: templating, DOM, events, compilation. This sort of leaves it stranded in 2014 while imposing large performance and size overheads.
preact Compatible with React, but creates much smaller and faster code. As a web-only framework, it cannot target native mobile applications, but can target Progressive Web Applications (PWA).
preact+htm Preact without compiling, building, packaging, or bundling. The code runs directly on the browser without any further tinkering. This makes developers very happy. Unfortunately, most of the React ecosystem assumes a packager, so they often do not work well here.
custom elements Bare metal embedding of logic into an HTML tag. Not a framework or library. It runs directly in the browser, so is compatible with any library or framework.
web components Like custom elements, but lets you write completely self-contained HTML tags. Once you get used to these, you may never want to go back!

These are independent: any one can be open at any time
